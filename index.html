World Map Feature Update

ShipBob's landing page aesthetically describes the service offering in a concise, component-based way. Each section is a progressive narrative that helps the user understand what service ShipBob provides. An important feature of the site is the world map section. It offers the first notice to the user that ShipBob is a regional and global brand - not just operating in one or two states.

The current world map on the live site is a static, onclick-based interaction. The user has to manually click on each button to identify the fulfillment centre location. Further to this, there is no exit-based action as the user has to click again to hide the location pin*.

Interactivity promotes usability

The updated world map now has two changes, an aesthetic change and an interactive change. Firstly, animation has been applied to all icons. This 'pulsing' animation offers varied animation-delays in order to maximise user attention. Unlike static images, the user is more likely to scan the entire map due to the visual aid of the animations. This then leads to interactivity.

The original feature triggers an event based on a click. The updated feature now triggers the event as soon as the user hovers over each icon. This will then display the corresponding location pin. Once the user has left the icon, the pin retracts back on exit in an animated fashion This method then prevents multiple location pins polluting the map and could possibly increase dwell time on the actual feature.

The purpose behind the update is to tell a new user story that ShipBob is 'constantly moving'. The use of animations aids the rhetoric that the fulfillment centres are 'active' and 'online'.
